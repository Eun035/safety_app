## 1. 라이드 서머리 → SNS 공유 카드 (신규)

실제 주행 이력만으로 "공유하고 싶은 카드"를 만들어 친구 유입을 유도하는 기능.

### 1.1 목업 제거 → 실데이터
| 이전 (목업) | 이후 (실주행 이력) |
|---|---|
| 하드코딩 SVG 곡선 | 실제 GPS 경로 스케치 라인 (시작=보라·끝=강조점) |
| "Sunset Boulevard" 고정 | 실제 목적지명 + `GPS N포인트` 뱃지 |
| Unsplash 가짜 사진 4장 | **Ride Highlights** — 최고속도·평균속도·안전점수 |
| "Chill AF ★★★★★" 고정 | 급정거 횟수 기반 실데이터 vibe |
| CO2 `2.4` 폴백 | 실제 계산값 |

- `src/utils/routeSketch.js` — GPS 경로 `[{lat,lng}]` → 종횡비 보존 SVG/캔버스 좌표 (경도 cos(lat) 보정, 중앙 정렬)
- `useRideSession.endRideSession` — summary에 실제 `path`/`destination` 포함

### 1.2 공유 카드 검은색 → Canvas 직접 렌더
- 증상: 비율 설정 후 공유 이미지가 **전부 검정**. `skipFonts`·워밍업·오프스크린 opacity 등 옵션을 바꿔도 지속.
- 원인: **html-to-image(SVG foreignObject) 방식**을 일부 안드로이드 브라우저/WebView가 이미지로 못 읽어 빈 래스터 생성. (중간에 캡처 대상 노드에 `opacity:0`을 건 자체 버그도 있었음.)
- 해결: html-to-image 제거 → **`src/utils/renderShareCard.js`에서 Canvas 2D API로 직접 렌더**(배경·경로·지표·CTA). 모든 브라우저에서 안정 동작.

### 1.3 디자인 테마(바이브) 4종
- `Neon`(다크·라임) / `Aurora`(다크·시안퍼플) / `Sunset`(웜·오렌지핑크) / `Minimal`(라이트·모노)
- `suggestTheme(급정거수)` — 안정=Aurora / 스무스=Neon / 다이내믹=Sunset 자동 추천, 칩으로 직접 변경(미리보기 경로색도 즉시 반영)

### 1.4 QR 제거 → 공유 아이콘 + 캡션 공유
- 카드의 QR 삭제, 하단을 **다운로드 CTA + 주소 + REF 코드 + 공유 아이콘 배지**로 교체
- 공유 시 **문구 + 해시태그 + 추천 링크** 캡션을 조합하고, 인스타처럼 텍스트를 무시하는 앱 대비 **클립보드 자동 복사**(붙여넣기용)

### 1.5 공유 버튼 동작 불가 수정
- 원인: `navigator.share()` 호출 **전에** `await clipboard.writeText()`가 사용자 제스처 활성화를 소모 → 공유 시트가 조용히 실패
- 해결: **카드 이미지 사전 렌더** 후 클릭 시 **`navigator.share`를 먼저** 호출, 클립보드 복사는 그 뒤(best-effort)로 이동

## 2. 목적지 입력(RideStartScreen) UX

- **음성 우선**: 화면 열릴 때 자동 포커스 제거(키보드 X) → 마이크로 먼저 입력. 안내 문구 추가.
- **실패 대비 키보드 전환**: 음성 인식 중 취소(X) 또는 입력 지우기 → 입력창 포커스 → 키보드 활성화.
- **출발지 기준 "자주 가는 곳"**: `src/utils/frequentDestinations.js` — 실제 선택 이력을 좌표 기준으로 누적(localStorage), 출발지 기준 ETA로 표시. 이력 없으면 큐레이션 폴백.
- **키보드 가림 수정**: `VisualViewport`로 키보드 높이 측정 → 시트를 키보드 위로 올려 하단 "자주 가는 곳"이 스크롤로 전부 노출.

## 6. 남은 것 / 참고

- 인앱 브라우저(카톡·인스타 내부)는 `navigator.share` 부재로 공유 시트가 안 뜰 수 있음 → 기본 브라우저에서 정상.
- 과거 주행 기록은 목적지 좌표가 없어 "자주 가는 곳"에 소급 반영 불가 → **선택 시점부터** 누적(향후 실데이터화).
- PWA 서비스워커 캐시로 이전 화면이 보일 수 있음 → 검증은 시크릿 창 권장.
