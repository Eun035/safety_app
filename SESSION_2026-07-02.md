## 1. 추천(Referral) 기능 완성

### 1.1 Supabase 백엔드 적용
- `supabase_referrals.sql` 실행 — `profiles` 컬럼(referral_code, referred_by_code), `referrals` 테이블 + RLS 3종, `grant_referral_reward` RPC.
- **RLS UPDATE 정책 패치**: `USING (invitee_user_id = auth.uid() OR invitee_user_id IS NULL)` — 익명 유저가 아직 주인 없는 landing 행을 클레임 가능하도록 (기존엔 NULL 행이 안 보여 연결이 조용히 실패).
- **`grant_referral_reward` RPC 재작성**: 기존엔 `referrals.invitee_user_id` 기준이라 클라이언트 링크 실패 시 보상이 안 나감 → **`profiles.referred_by_code` 기준**으로 변경(클라가 이건 성공적으로 채움). `referrals` 행은 RPC가 self-heal(청구/생성), 멱등성은 `reward_granted`로 보장.

### 1.2 검증 (E2E)
- `?ref=B5188C`로 친구(B) 유입 → `profiles.referred_by_code` 저장 확인
- `SELECT grant_referral_reward(invitee_id)` → 초대자·피초대자 **양쪽 +500P** 확인
- 재호출 시 `already_rewarded` 반환 → **중복 지급 방지** 확인

### 1.3 발견한 클라 이슈 (비차단)
- `App.jsx`의 `referrals` UPDATE에 붙은 `.order().limit()`이 서버에서 안 먹어 `invitee_user_id` 저장이 조용히 실패. 단, RPC를 `profiles.referred_by_code` 기준으로 바꿔 보상은 정상 작동 → blocking 아님.

## 2. `/r/` 짧은 링크 404 → `?ref=` 우회

- 프로덕션에서 `/r/{code}` 및 모든 sub-path가 Vercel **404**. `vercel.json`에 SPA catch-all rewrite가 있고 배포 커밋에도 포함돼 있는데도 미적용 (원인 미상 Vercel 이슈).
- **해결**: `buildReferralUrl`을 `/r/${code}` → `/?ref=${code}` 로 변경. 루트 `/`는 index.html을 직접 서빙하고 `captureReferralFromUrl`이 `?ref`도 파싱하므로 서버 rewrite에 의존하지 않음. (커밋 `6315375`)

## 3. 음성(TTS) 개선 및 진단

### 3.1 괄호·슬래시 병기 정리 (`sanitizeForSpeech`)
- [useVoiceGuidance.js](src/hooks/useVoiceGuidance.js)에 `sanitizeForSpeech()` 추가 → `speak()` 발화 직전 적용.
- 규칙: 괄호 병기 제거(`인도(보도)`→`인도`, 전각 포함), 단어/단어 슬래시는 앞쪽만(`인도/보도`→`인도`, `km/h`→`km`, `QR/NFC`→`QR`), 숫자 슬래시(`3/4`·`2020/12`)는 유지.
- **화면 표기는 원본 유지, 읽을 때만** 정리. 모든 언어·모든 speak 호출에 자동 적용.

### 3.2 음성 무음 진단 (앱 버그 아님)
- 노트북에서 음성이 안 난 원인 = **기기 자체 오디오 출력 문제**(다른 소리도 안 남). 한국어 음성(Microsoft Heami)·코드 모두 정상. 조치 불필요.

## 4. 다국어(i18n) 전면 전환 — 핵심 작업

react-i18next 기반, `src/locales/i18n.js` 단일 사전. 언어 **ko / en / ja / zh-CN**. 언어 미설정 문구는 폴백.

### 4.1 인프라/패턴
- 대부분 `t("key")` 방식. 정적 데이터(랜드마크·거점 이름·배지 등)는 **literal-key** 방식(`t(값, {defaultValue})` + ja/zh/en만 키 추가, ko는 defaultValue).
- **결정(사용자)**: 지도 주소 등 정적 데이터만 번역, Kakao 실시간 검색결과·현위치 주소는 한국어 유지. 코드 검색 쿼리어(`fetchPlaces('공원')` 등)도 한국어 유지.
