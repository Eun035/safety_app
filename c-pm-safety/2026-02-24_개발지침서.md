# C-Safe 개발 지침서 (2026-02-24 기준)

본 문서는 C-Safe 애플리케이션의 핵심 비즈니스 로직(B2G/O2O 연계) 및 보안 강화를 위해 새롭게 개발 및 연동된 내용들의 지침을 요약합니다. 향후 유지보수 및 확장을 위해 각 모듈의 동작 방식을 기술합니다.

---

## 1. 지식재산 보호 및 코드 난독화 (빌드 프로세스)
경쟁사로의 핵심 로직 유출을 방지하기 위해 배포(Production) 빌드 시 소스코드가 강력하게 난독화됩니다.
*   **관련 파일**: `vite.config.js`
*   **변경 사항**: 기존 `vite-plugin-obfuscator` 오류(Vite 6 / Rolldown 호환성 문제)를 해결하기 위해, `javascript-obfuscator` 모듈을 직접 호출하는 커스텀 Rollup 플러그인(`obfuscatorPlugin`)을 인라인으로 작성했습니다.
*   **주요 설정**:
    *   `compact: true`
    *   `controlFlowFlattening: true` (제어문 평탄화)
    *   `stringArray: true`, `stringArrayShuffle: true` (문자열 난독화 및 셔플링)
    *   `identifierNamesGenerator: 'hexadecimal'` (식별자를 16진수로 변경)
*   **운영 지침**: 차기 유지보수 시 콘솔 디버깅이 필요할 수 있으므로, 난독화 로직은 `npm run build` 스크립트를 통해서만 동작하도록 구성되었습니다. (개발 서버 환경에서는 작동하지 않음)

---

## 2. 천안사랑카드 포인트 연동 (O2O 헬멧 스테이션)
기존 "10% 할인 쿠폰" 지급 로직을 **"천안사랑카드 포인트" 지급 메커니즘**으로 전면 개편했습니다.
*   **관련 파일**: `App.jsx`, `components/common/QRScanner.jsx`, `components/common/CouponBox.jsx`
*   **로직 변경**:
    *   `QRScanner` 스캔 성공 시 할인 쿠폰이 아닌 `amount: '+500P', type: '천안사랑카드'` 속성을 지닌 **포인트 객체**를 생성합니다.
    *   변경된 `CouponBox` (현 UI 명칭: '천안사랑카드 적립 내역') 컴포넌트에서 지급된 내역을 리스트업하며, 현재 사용자의 누적 포인트와 연동되는 인터페이스를 제공합니다.

---

## 3. 천안시 PM 주차장 데이터 연동 및 마커 시각화
천안시에서 제공한 410여 개의 PM 주차장 엑셀 데이터를 애플리케이션 지도 상에 시각화합니다.
*   **데이터 파셔 스크립트 (`parse_parking.js`)**:
    *   세 개의 원본 TXT 파일 내의 좌표 데이터를 정규화하여 `lat`, `lng`, `locationName`, `capacity` 속성을 지닌 JSON 배열로 파싱 및 출력합니다.
    *   출력된 데이터는 `src/data/pm_parking_data.json` 하위에 정적으로 저장됩니다.
*   **데이터 연동 훅 (`usePMParkingData.js`)**:
    *   파싱 완료된 JSON 파일을 React 전역 공간으로 가져올 수 있도록 커스텀 훅을 통해 제공합니다.
*   **지도 시각화 (`MapContainer.jsx`)**:
    *   `CustomOverlayMap`을 통해 데이터를 순회하며 **둥근 민트색 'P' 마커**를 카카오맵 위에 일괄 렌더링합니다.
    *   사용자가 해당 마커를 클릭 시 지정된 주차구역 명칭과 주차가능 대수가 포함된 음성 안내 및 툴팁을 활성화합니다.

---

## 4. '운행 종료' 하버사인 지오펜싱 및 리워드 처리 로직 
합법 주차를 장려하기 위해 사용자 기기 GPS와 PM 주차 구역 간의 거리를 판별합니다.
*   **관련 파일**: `App.jsx`, `utils/distance.js`
*   **거리 계산 모듈 (`calculateDistance`)**: Haversine 공식을 사용해 두 좌표(`lat`, `lng`) 간의 직선 거리를 미터(m) 단위로 반환합니다.
*   **종료 지오펜싱 로직 (`App.jsx` 내 `endRide` 핸들러)**:
    1.  사용자가 지도 하단의 **운행 종료** 버튼을 클릭 시 트리거됩니다.
    2.  `pmParkingData.json`를 순회하며 사용자의 최종 측정 위치 좌표와의 최단거리(`minDistance`)를 계산합니다.
    3.  **최단거리가 10m 이하일 경우 (성공)**: `[+100P 적립 완료]` 시각적 모달과 함께 "주차 확인 성공. 리워드 100포인트가 적립되었습니다." 라며 성공 보상을 처리합니다.
    4.  **최단거리가 10m 초과할 경우 (경고)**: 불법 구역으로 인식하여 "경고. 지정된 주차 구역이 아닙니다. 불법 주차 시 페널티가 부과될 수 있습니다."라는 빨간색 경고 모달 창과 음성 안내를 시연합니다.

## 5. 탑승 의무 안전 퀴즈 랜덤 노출 및 초기화 기능
최초 실행 시 반드시 거쳐야 하는 의무 교육(Safety Quiz)의 사용자 경험을 개선했습니다.
*   **관련 파일**: `components/quiz/SafetyQuiz.jsx`, `App.jsx`
*   **다이나믹 퀴즈 생성**: 총 9문항의 전체 문제 은행에서 렌더링 시점에 **무작위 시드(Random Seed)로 3개의 문항만 추출**하도록 배열 슬라이싱을 적용했습니다.
*   **상태 저장 및 리셋(디버깅) 기능**:
    *   성공적인 통과 여부는 사용자의 로컬스토리지(캐시)인 `csafe_onboarding_completed_v5` 속성에 저장되어 이후 렌더링 화면에서 생략됩니다.
    *   다만 개발 시연 목적으로 로컬 스토리지를 삭제하고 퀴즈 창을 부활시키기 위해 앱의 로고/컨트롤 컨테이너 하단에 **붉은색 'RESET (온보딩 초기화)' 버튼**을 배치하였습니다. 클릭 시 온보딩 로컬 캐시가 즉시 삭제되고 스플래시 인트로 화면으로 복구됩니다.
